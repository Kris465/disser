# ИСР и расписание проекта

**Версия:** 6.0
**Дата:** 16 мая 2026

## 1. Иерархическая структура работ

```
1.0 Инициация проекта
   1.1 Разработка устава проекта
   1.2 Идентификация заинтересованных сторон
   1.3 Утверждение устава

2.0 Планирование
   2.1 Сбор и анализ требований
       2.1.1 Изучение существующей системы изолированных сессий
       2.1.2 Идентификация параметров защищённости
       2.1.3 Документирование требований
   2.2 Разработка модели угроз
       2.2.1 Идентификация активов
       2.2.2 Идентификация угроз
       2.2.3 Оценка рисков
       2.2.4 Определение контрмер
   2.3 Проектирование базы знаний
       2.3.1 Разработка онтологии домена
       2.3.2 Формализация правил (IF-THEN)
       2.3.3 Определение весов и множителей
   2.4 Проектирование архитектуры
       2.4.1 Определение компонентов
       2.4.2 Проектирование потоков данных
       2.4.3 Выбор технологического стека
   2.5 Разработка плана тестирования
       2.5.1 Определение эталонных стендов
       2.5.2 Определение метрик качества
       2.5.3 Разработка тест-кейсов
   2.6 Разработка реестра рисков
   2.7 Разработка расписания

3.0 Исполнение
   3.1 Реализация модуля сбора данных
       3.1.1 Парсер конфигураций Docker
       3.1.2 Парсер сетевых настроек
       3.1.3 Парсер конфигурации аутентификации
   3.2 Реализация базы знаний
       3.2.1 Реализация онтологии
       3.2.2 Загрузка правил из конфигурационного файла
       3.2.3 Реализация матрицы весов
   3.3 Реализация механизма вывода
       3.3.1 Реализация forward chaining
       3.3.2 Реализация расчёта индекса
       3.3.3 Реализация классификации вердиктов
   3.4 Реализация интерфейса
       3.4.1 Dashboard с цветовым кодом
       3.4.2 Детализация по правилам
       3.4.3 Экспорт отчётов

4.0 Мониторинг и контроль
   4.1 Тестирование на эталонных стендах
       4.1.1 Тест Perfect Config
       4.1.2 Тест Default Config
       4.1.3 Тест Broken Config
   4.2 Расчёт метрик качества
       4.2.1 Precision, Recall, F1
       4.2.2 Сравнение с целевыми значениями (≥95%)
   4.3 Валидация экспертом ИБ
   4.4 Контроль изменений (Change Request)

5.0 Закрытие
   5.1 Документирование
       5.1.1 Руководство администратора
       5.1.2 Итоговый отчёт
   5.2 Формальная приёмка
   5.2.1 Демонстрация спонсору
   5.2.2 Подписание акта приёмки
```
