# Проект базы знаний экспертной системы

**Версия:** 7.0
**Дата:** 16 мая 2026

## 1. Онтология домена

### 1.1. Классы параметров

База знаний оперирует следующими классами параметров безопасности:

**Сессия (Session)** — параметры изолированной рабочей сессии:

| Параметр | Тип | Возможные значения | Описание |
|----------|-----|-------------------|----------|
| session_type | enum | VDI, RDP, Container | Тип сессии |
| isolation_level | enum | NONE, WEAK, MODERATE, STRONG | Уровень изоляции |
| clipboard_isolation | boolean | TRUE, FALSE | Изоляция буфера обмена |
| disk_redirection | boolean | TRUE, FALSE | Перенаправление дисков |
| screen_capture_protection | boolean | TRUE, FALSE | Защита от захвата экрана |

**Аутентификация (Authentication)** — параметры контроля доступа:

| Параметр | Тип | Возможные значения | Описание |
|----------|-----|-------------------|----------|
| mfa_enabled | boolean | TRUE, FALSE | Наличие MFA |
| password_policy | enum | WEAK, MODERATE, STRONG | Парольная политика |
| auth_logging | boolean | TRUE, FALSE | Логирование аутентификации |

**Контейнер (Container)** — параметры безопасности Docker-контейнера:

| Параметр | Тип | Возможные значения | Описание |
|----------|-----|-------------------|----------|
| cap_drop_all | boolean | TRUE, FALSE | Отзыв всех capabilities |
| read_only_rootfs | boolean | TRUE, FALSE | Корневая ФС только для чтения |
| no_new_privileges | boolean | TRUE, FALSE | Запрет повышения привилегий |
| auto_remove | boolean | TRUE, FALSE | Автоудаление контейнера |
| tmpfs_enabled | boolean | TRUE, FALSE | Использование tmpfs для /tmp, /run |

**Сеть (Network)** — параметры сетевой безопасности:

| Параметр | Тип | Возможные значения | Описание |
|----------|-----|-------------------|----------|
| vlan_segmented | boolean | TRUE, FALSE | Сегментация VLAN |
| firewall_enabled | boolean | TRUE, FALSE | Межсетевой экран |
| traffic_encrypted | boolean | TRUE, FALSE | Шифрование трафика |

**Мониторинг (Monitoring)** — параметры наблюдения:

| Параметр | Тип | Возможные значения | Описание |
|----------|-----|-------------------|----------|
| audit_logging | boolean | TRUE, FALSE | Аудит событий безопасности |
| anomaly_detection | boolean | TRUE, FALSE | Обнаружение аномалий |

## 2. Правила вывода

### 2.1. Формат правил

Каждое правило имеет структуру:

```
IF (<параметр1> = <значение1>) AND (<параметр2> = <значение2>) ...
THEN:
  Риск = <CRITICAL | HIGH | MEDIUM | LOW>
  Вес = <число>
  Угроза = <Txx>
  Сообщение = <текст>
  Рекомендация = <текст>
```

### 2.2. Категория: Изоляция сессии

| ID | Правило | Условия | Риск | Вес | Угроза |
|----|---------|---------|------|-----|--------|
| S01 | Утечка через буфер обмена | clipboard_isolation=FALSE AND isolation_level∈{NONE,WEAK} | CRITICAL | 10 | T01 |
| S02 | Перенаправление дисков | disk_redirection=TRUE | HIGH | 8 | T02 |
| S03 | Нет защиты от захвата экрана | screen_capture_protection=FALSE | MEDIUM | 5 | T07 |

**Рекомендации:**
- S01: Включить изоляцию буфера обмена в настройках сессии
- S02: Отключить перенаправление дисков для изолированных сессий
- S03: Включить защиту от скриншотов в политиках сессии

### 2.3. Категория: Аутентификация

| ID | Правило | Условия | Риск | Вес | Угроза |
|----|---------|---------|------|-----|--------|
| A01 | MFA не требуется | mfa_enabled=FALSE | HIGH | 9 | T03 |
| A02 | Слабая парольная политика | password_policy=WEAK | MEDIUM | 6 | T03 |
| A03 | Нет логирования аутентификации | auth_logging=FALSE | MEDIUM | 5 | T10 |

**Рекомендации:**
- A01: Включить двухфакторную аутентификацию для всех типов сессий (VDI, RDP, Container)
- A02: Усилить парольную политику (мин. длина, сложность, ротация)
- A03: Включить логирование всех событий входа/выхода

### 2.4. Категория: Контейнер

| ID | Правило | Условия | Риск | Вес | Угроза |
|----|---------|---------|------|-----|--------|
| C01 | Избыточные привилегии | cap_drop_all=FALSE | HIGH | 8 | T04 |
| C02 | Записываемая корневая ФС | read_only_rootfs=FALSE | MEDIUM | 6 | T04 |
| C03 | Возможна эскалация привилегий | no_new_privileges=FALSE | HIGH | 7 | T04 |
| C04 | Нет автоудаления | auto_remove=FALSE | LOW | 3 | T04 |

**Рекомендации:**
- C01: Добавить `--cap-drop=ALL` в параметры запуска
- C02: Добавить `--read-only` и `--tmpfs` для /tmp, /run
- C03: Добавить `--security-opt=no-new-privileges:true`
- C04: Добавить `--rm` в параметры запуска

### 2.5. Категория: Сеть

| ID | Правило | Условия | Риск | Вес | Угроза |
|----|---------|---------|------|-----|--------|
| N01 | Нет сегментации VLAN | vlan_segmented=FALSE | HIGH | 8 | T09 |
| N02 | Firewall отключён | firewall_enabled=FALSE | HIGH | 9 | T09 |
| N03 | Трафик не шифруется | traffic_encrypted=FALSE | HIGH | 8 | T08, T05 |
| N04 | Нет защиты протокола удалённого доступа | traffic_encrypted=FALSE AND session_type∈{VDI,RDP,Container} | HIGH | 7 | T05 |

**Рекомендации:**
- N01: Настроить VLAN для изоляции трафика сессий
- N02: Включить и настроить firewall
- N03: Включить TLS/SSL для всех соединений
- N04: Использовать TLS-туннелирование для VNC/RDP, ограничить доступ по IP/MAC

> **Примечание:** Угроза T05 (атака на протокол VNC/RDP) покрывается двумя правилами: N03 (шифрование трафика) и N04 (защита протокола). Правило N04 срабатывает при отсутствии шифрования для любого типа сессии, обеспечивая дополнительный контроль именно на уровне протокола удалённого доступа.

### 2.6. Категория: Мониторинг

| ID | Правило | Условия | Риск | Вес | Угроза |
|----|---------|---------|------|-----|--------|
| M01 | Нет аудита | audit_logging=FALSE | MEDIUM | 6 | T10 |
| M02 | Нет обнаружения аномалий | anomaly_detection=FALSE | LOW | 4 | T06 |

**Рекомендации:**
- M01: Включить логирование всех событий безопасности
- M02: Внедрить систему обнаружения аномалий поведения

## 3. Расчёт индекса защищённости

### 3.1. Формула

```
Максимальный балл (MAX) = Σ вес_правила = 102

Штраф = Σ (вес_сработавшего × множитель_риска)

Индекс = 100 - (Штраф / MAX) × 100
```

### 3.2. Множители риска

| Уровень риска | Множитель |
|---------------|-----------|
| CRITICAL | 1.0 |
| HIGH | 0.75 |
| MEDIUM | 0.5 |
| LOW | 0.25 |

### 3.3. Пример расчёта

**Ситуация:** сработали правила S01 (CRITICAL, w=10), A01 (HIGH, w=9), C02 (MEDIUM, w=6).

```
Штраф = 10×1.0 + 9×0.75 + 6×0.5 = 10 + 6.75 + 3 = 19.75
Индекс = 100 - 19.75/102 × 100 = 100 - 19.36 = 80.64
Вердикт: "Безопасно" (80 ≤ 80.64)
```

### 3.4. Пороги вердиктов

| Индекс | Вердикт | Цвет |
|--------|---------|------|
| 80–100 | Безопасно | 🟢 Зелёный |
| 50–79 | Требуется доработка | 🟡 Жёлтый |
| 0–49 | Уязвимо | 🔴 Красный |
