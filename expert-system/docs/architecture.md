# Архитектура экспертной системы оценки защищённости

**Версия:** 4.0
**Дата:** 16 мая 2026

---

## 1. Назначение

Экспертная система предназначена для автоматической оценки уровня защищённости сегмента сети, использующего модель изолированных сессий рабочих станций. Система является надстройкой над существующим проектом ООО «Академия ТОП» и использует его компоненты как источники данных.

## 3. Компоненты

### 3.1. Модуль сбора данных (Data Collection)

**Назначение:** автоматический сбор параметров конфигурации из компонентов системы изолированных сессий.

**Источники данных:**

| Источник | Параметры | Метод сбора |
|----------|-----------|-------------|
| Docker Engine | cap_drop, read_only, security_opt, auto_remove | `docker inspect` |
| Сетевые настройки | VLAN, firewall, шифрование | `iptables -L`, конфигурационные файлы |
| Аутентификация | MFA, парольная политика, логи | API сервера аутентификации, GPO |
| Мониторинг | аудит, обнаружение аномалий | Конфигурация системы мониторинга |

**Выход:** словарь фактов `{параметр: значение}`.

### 3.2. База знаний (Knowledge Base)

**Назначение:** хранение формализованных экспертных знаний о защищённости изолированных сессий.

**Структура:**

- **Онтология домена** — перечень всех параметров, их типы и допустимые значения
- **Правила** — набор правил IF-THEN (15 правил в 6 категориях)
- **Матрица весов** — числовые веса и множители риска для каждого правила

**Формат хранения:** YAML/JSON-файл, что позволяет редактировать правила без перекомпиляции.

### 3.3. Механизм логического вывода (Inference Engine)

**Назначение:** применение правил базы знаний к собранным фактам и расчёт итоговой оценки.

**Алгоритм (Forward Chaining):**

```
1. Получить факты F от модуля сбора данных
2. triggered_rules = []
3. FOR each rule R в базе знаний:
     a. Для каждого условия C в R.conditions:
          - Извлечь значение параметра из F
          - Сравнить с ожидаемым значением
     b. Если ВСЕ условия выполнены:
          - Добавить R в triggered_rules
4. Рассчитать индекс защищённости
5. Определить вердикт
6. Сформировать отчёт
```

**Расчёт индекса:**

```
penalty = Σ (weight_i × risk_multiplier_i) для всех triggered_rules
index = max(0, 100 - penalty / max_score × 100)
```

### 3.4. Интерфейс (Dashboard)

**Назначение:** визуализация результатов оценки.

**Функции:**

- Отображение индекса защищённости (числовое значение + цветовая индикация)
- Детализация по сработавшим правилам
- Рекомендации по устранению уязвимостей
- Экспорт отчёта (PDF/HTML)

---

## 4. Поток данных

```
1. [Сбор]   Модуль сбора данных → Факты (словарь параметров)
2. [Вывод]  Inference Engine + База знаний → Сработавшие правила
3. [Расчёт] Сработавшие правила → Индекс защищённости + Вердикт
4. [Отчёт]  Результат → Dashboard / Экспорт
```

## 5. Интеграция с существующей системой

Экспертная система интегрируется с компонентами проекта ООО «Академия ТОП»:

| Компонент ЭС | Интеграция с |
|-------------|-------------|
| Docker Parser | Docker Engine на рабочих станциях |
| Network Parser | Сетевое оборудование филиала |
| Auth Parser | Сервер аутентификации / Active Directory |
| Dashboard | Сервер управления (FastAPI) |

## 6. Технологический стек (предлагаемый)

| Компонент | Технология | Обоснование |
|-----------|-----------|-------------|
| Язык реализации | Python 3.10+ | Единый стек с основным проектом |
| База знаний | YAML/JSON | Человекочитаемый формат, легко редактировать |
| Inference Engine | Собственная реализация | Простота, прозрачность логики |
| UI | FastAPI + Jinja2 + HTML | Локальный веб-интерфейс, единый стек с сервером управления |
| Отчёты | Jinja2 → HTML → PDF | Генерация отчётов из шаблонов |
| Тестирование | unittest/pytest | Стандартные инструменты |
