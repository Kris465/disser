# Модель угроз для изолированных сессий рабочих станций

**Версия:** 2.0
**Дата:** 16 мая 2026

---

## 1. Область применения

Модель угроз разработана для сегмента сети, использующего модель изолированных сессий рабочих станций (проект ООО «Академия ТОП»). Сегмент включает:

- **Сервер управления** (Docker Registry + FastAPI)
- **Рабочие станции** (Windows 10/11 + Docker Desktop + лаунчер)
- **Сеть** (локальная сеть филиала, VLAN, firewall)

---

## 2. Активная модель

### 2.1. Активы

| ID | Актив | Ценность | Владелец |
|----|-------|----------|----------|
| A1 | Конфиденциальные данные пользователей (документы, учётные записи) | Высокая | Организация |
| A2 | Конфигурации безопасности контейнеров | Высокая | Администратор |
| A3 | Образы Docker (designer-base, developer-base) | Средняя | Администратор |
| A4 | Логи аутентификации и аудита | Средняя | Специалист ИБ |
| A5 | Сетевая инфраструктура (VLAN, firewall) | Высокая | Сетевой администратор |

### 2.2. Угрозы

| ID | Угроза | Категория | Актив | Описание |
|----|--------|-----------|-------|----------|
| T01 | Утечка через буфер обмена | НСД | A1 | Копирование данных из изолированной сессии на хостовую ОС через общий clipboard |
| T02 | Перенаправление дисков | НСД | A1, A2 | Монтирование хостовых дисков в контейнер, запись данных на хост |
| T03 | Компрометация учётных данных | Аутентификация | A1 | Подбор/кража паролей при отсутствии MFA |
| T04 | Эскалация привилегий в контейнере | Контейнер | A2 | Выход за пределы изоляции при недостаточных security options |
| T05 | Атака на протокол VNC/RDP | Сетевой | A1 | Перехват или инъекция в трафик удалённого доступа |
| T06 | Кейлоггер в сессии | Вредоносное ПО | A1 | Перехват ввода пользователя внутри сессии |
| T07 | Захват экрана | НСД | A1 | Несанкционированный скриншот или запись экрана сессии |
| T08 | Прослушивание сети | Сетевой | A1 | Перехват незашифрованного трафика между станцией и сервером |
| T09 | Межсегментный доступ | Сетевой | A1, A5 | Доступ между сегментами при некорректной VLAN/firewall конфигурации |
| T10 | Отсутствие аудита | Мониторинг | A4 | Невозможность расследования инцидентов при отключённом логировании |
